## 學習階段劃分

### 階段一：基礎建立（Week 1-3）
建立演算法分析能力與核心資料結構基礎
- Week 1: 演算法複雜度分析 + Arrays
- Week 2: Linked Lists + Stack + Queue
- Week 3: Hash Table + Binary Search + Bitwise Operations

### 階段二：樹與排序（Week 4-6）
掌握樹狀結構與各種排序算法
- Week 4: Trees 基礎 + Binary Search Trees
- Week 5: Heap/Priority Queue + Sorting 基礎
- Week 6: Sorting 進階實作

### 階段三：圖論（Week 7-8）
學習圖的表示與核心算法
- Week 7: Graphs 基礎（BFS, DFS, 圖的表示）
- Week 8: Graphs 進階（最短路徑, MST, 進階算法）

### 階段四：遞迴與動態規劃（Week 9-10）
掌握遞迴思維與動態規劃解題技巧
- Week 9: Recursion + Dynamic Programming 入門
- Week 10: Dynamic Programming 深入練習

### 階段五：設計模式（Week 11）
複習與強化設計模式知識（利用既有經驗）
- Week 11: Design Patterns

### 階段六：進階主題（Week 12-14）
擴展知識廣度，涵蓋系統與進階算法
- Week 12: Combinatorics, NP-Complete, 系統知識
- Week 13: Testing, String Algorithms, Tries
- Week 14: Networking 與其他剩餘主題

### 階段七：總複習與面試準備（Week 15-16）
整合所學，準備面試
- Week 15: Final Review + 核心主題複習
- Week 16: Resume 更新 + 面試技巧 + Mock Interviews

## 學習方法建議

### 影片學習（40-50%時間）
- 專注理解概念，不必記住所有細節
- 做筆記記錄關鍵概念
- 可以 1.5x 速度觀看熟悉的內容

### 程式實作（40-50%時間）
- **必須**親手實作每個資料結構
- 使用 Java 實作（利用你的優勢）
- 同時也嘗試用 Python 實作基礎資料結構（許多公司面試允許 Python）
- 每個實作都要測試邊界條件

### LeetCode 練習（每週至少 5-10 題）
- 選擇與當週主題相關的題目
- 優先練習 Easy 和 Medium 難度
- 記錄解題思路和時間複雜度分析

### 每週檢視（週五下午）
- 回顧本週學習內容
- 確認所有 checkbox 都已完成
- 準備下週學習計劃

## 注意事項

1. **不要過度追求完美**：重點是理解核心概念，而非記住所有細節
2. **實作比理論重要**：確保每個資料結構都親手實作過
3. **定期複習**：每 2-3 週複習之前的內容
4. **保持健康**：每天 7 小時是高強度學習，注意休息
5. **彈性調整**：如果某週進度超前或落後，可以調整後續計劃
